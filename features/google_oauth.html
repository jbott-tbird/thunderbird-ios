Thunderbird iOS

Google OAuth Findings

Bottom Line

Google Cloud configuration is necessary, but it is not enough by itself. The app already has the start of an OAuth flow, but Gmail OAuth cannot work until the app exchanges the authorization code for real tokens, stores refreshable credentials, and teaches IMAP/SMTP to authenticate with XOAUTH2.

Google Setup

  1. Create an iOS OAuth client ID in Google Cloud for bundle ID net.thunderbird.ios. The OAuth client type needs to be iOS, not Android or Web.
  2. Add the reversed iOS client ID URL scheme to Thunderbird/Thunderbird/Info.plist. The current scheme is net.thunderbird.android.debug, which does not match an iOS Google client.
  3. Use Gmail's IMAP/SMTP OAuth scope: 
    https://mail.google.com/

Verification Risk

The Gmail IMAP/SMTP scope https://mail.google.com/ is broad mailbox access. A public app using this scope will need Google OAuth verification and may require a security assessment before broad production use.

Current Code Blockers

  • OAuthButton.swift starts ASWebAuthenticationSession, but ignores the callback URL and currently writes a fake bearer token.
  • Core/Sources/Autoconfiguration/URLSession.swift has URLSession.token(...) stubbed with fatalError().
  • Core/Sources/Account/Server.swift explicitly rejects .oAuth2 for IMAP and SMTP bridge setup.

Implementation Path

  1. Replace the hardcoded Google client ID in OAuth2.swift with the real iOS OAuth client ID.
  2. Change the Google redirect URI to the reversed client ID scheme: 
    com.googleusercontent.apps.YOUR_IOS_CLIENT_ID_PREFIX:/oauth2redirect
  3. Add PKCE by generating a code_verifier, sending code_challenge and code_challenge_method=S256 on the auth URL, then sending code_verifier in the token exchange.
  4. Parse the callback URL from ASWebAuthenticationSession, extract code, and POST it to https://oauth2.googleapis.com/token.
  5. Store both access_token and refresh_token. Gmail access tokens are short-lived, so the app needs refresh support rather than a single bearer string.
  6. Implement Gmail XOAUTH2 for IMAP and SMTP. Gmail expects SASL XOAUTH2 with an access token, not normal password login.
