Native Google clients have no secret — PKCE binds the auth request to the token exchange1

IMAP/AuthenticateCommand.swift 🔐+77 SASL XOAUTH2 IMAP auth. Builds the initial response user=<email>^Aauth=Bearer <token>^A^A and a NIO handler for the exchange.On a bad/expired token Gmail sends a base64 challenge before the tagged NO — the handler treats that as a fast auth failure rather than waiting for the command to time out. What else would help you understand these changes

✓ Already captured here Changed-vs-new split, status, line delta, the phase each file belongs to, and a per-file breakdown of the larger changes — so the grid doubles as a map of the Phase 1–9 build order.
  • Risk / security review focus — the 🔐 rows (the client_….plist credentials, PKCE, Token, XOAUTH2 auth, the Info.plist redirect scheme) are where a reviewer should spend time, and tie directly to oauth_audit.html. Worth confirming the committed client plist is intended to be tracked.
  • Test-coverage gaps — Core parsing/data code got unit tests, but MessageManager, Inbox, Outbox, and the SwiftUI views are exercised only by manual runs against the Gmail test account.

  • Net code movement — the −334/−320/−263 deletions (TempEmailModel, ReadEmailView, EmailCellView) are the mock → real-data migration, not feature loss; read them alongside the new model files.
