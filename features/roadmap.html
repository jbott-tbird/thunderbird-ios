Phase 3 · Compose & Direct Send

A new-message composer that sends through Gmail over SMTP XOAUTH2. The pattern mirrors Phase 2's IMAP XOAUTH2 work, applied to the SMTP layer — which already sends (SMTPClient.send) but currently authenticates with AUTH LOGIN only.

🎯 Locked scope New messages only (reply/forward is Phase 4). Direct send now — compose → send → inline result; the persisted outbox/queue with drafts & retry is Phase 7.

Teach the SMTP client to authenticate with a bearer token, the same SASL mechanism IMAP already uses.

  1. AUTH XOAUTH2 request — add a case to RequestEncoder emitting AUTH XOAUTH2 <base64>, where the payload is the SASL string user=<email>^Aauth=Bearer <token>^A^A (same as IMAP; ^A = U+0001).
  2. SendHandler branch — when the server is OAuth2, run the XOAUTH2 exchange instead of AUTH LOGIN; handle Gmail's 235 success / 334 error-challenge.
  3. SMTP.Server auth flag — add an Authentication enum (mirrors the one added to IMAP.Server in Phase 2).
  4. Accept OAuth2 — stop Account/Server.swift rejecting .oAuth2 for SMTP; pass the token as the credential and set the flag.
  5. Refresh the outgoing token — extend token refresh to the outgoing server before send. Today Account.refreshTokenIfNeeded only refreshes the incoming server's keychain entry; SMTP has a separate entry.
🔌Connection Gmail SMTP is smtp.gmail.com:587 with STARTTLS, which the existing send pipeline already supports.

A small service that turns an account + a composed message into a sent email.

  • MessageManager.send(_:) (or a SendManager) builds an SMTPClient from the account's outgoingServer and sends an SMTP.Email.
  • The send pipeline and the SMTP.Email model (sender / recipients / cc / bcc / subject / body) already exist — this is wiring, not new transport.

A presented composer wired to the (currently disabled) compose button in EmailListView.

  1. Fields — to / cc / bcc / subject / rich-text body (Infomaniak RichHTMLEditor) with a bold / italic / underline / list formatting bar.
  2. Build — assemble an SMTP.Email with the account identity as sender and a text/html MIME.Body from the editor's HTML.
  3. Send — call the send service; show inline sending / success / error state; dismiss on success.

Validation: the XOAUTH2 SASL building is unit-testable in Core; the composer + send verify in Xcode against the Gmail test account.

Phases 4–9

Done. ReadEmailView's reply / reply-all / forward buttons (bottom toolbar + sender menu) open the composer via a MessageDraft: Re:/Fwd: subject prefixing, reply-to-or-from target selection, reply-all Cc from the original recipients minus self, and an HTML-quoted original (cite blockquote for replies, forwarded-message header for forwards). Builds directly on Phase 3's ComposeView.

Done. MessageManager.monitorInbox() runs an IMAP IDLE loop on a dedicated connection (Account.newIMAPClient(), so it never blocks pooled fetches), re-selecting INBOX on each wake to fetch newly-arrived envelopes (InboxUpdate.added) or signal needsReconcile on removals, re-IDLEing on a 25-min window and reconnecting on BYE. Inbox.startLiveUpdates() streams pushes into the existing upsert; EmailListView starts it after the first load and falls back to pull-to-refresh when IDLE is unsupported.

Done. MessageBody.Attachment now carries its RFC 3501 section path + transfer encoding (the MIME walk tracks section numbers); IMAPClient.fetch(uid:section:) fetches just BODY[section] and Data.transferDecoded(_:) decodes base64 / quoted-printable to bytes. Email persists lightweight AttachmentInfo descriptors, and ReadEmailView lists attachments with a type icon + size; tapping downloads the bytes to a temp file and opens Quick Look.

Done. Compose hands messages to a SwiftData-backed OutgoingEmail queue and dismisses immediately; the Outbox manager drains it oldest-first over SMTP, deletes on success, and retries failures with capped exponential back-off (5 attempts), surviving relaunch and offline. OutboxView lists queued/sending/failed messages with swipe retry/delete, and EmailListView shows a count + entry point when the queue is non-empty.

Done. EmailListView tracks a persisted selectedAccountID, shows that account's INBOX (filtering the all-accounts @Query), and rebuilds the per-account Inbox + IDLE stream on switch; DrawerView gained an account switcher. ReadEmailView resolves the owning account by the message's own accountID (body load, attachments, reply/forward all use the right credentials), and ContentView's auth gate passes if any account is authorized.

Done. Refresh prunes messages deleted server-side (a fetched page is a contiguous UID window, so stored messages in that window absent from the page are removed; the newest page is authoritative at/above its lowest UID). Inbox.loadMore() pages in older messages via fetchInbox(count:olderThan:), and setRead/setFlagged/markAllRead push \Seen/\Flagged through IMAPClient.store(uid:flag:enabled:) — flags reconcile both ways. Scroll-to-end paginates; swipes toggle read/flag.
